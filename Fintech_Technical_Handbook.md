# 第一章 金融基础概念

## 1.1 股票是什么

**大白话**：股票就是一家公司的"所有权凭证"。比如你买了一股苹果公司（AAPL）的股票，你就拥有了苹果公司的极小一部分。股票价格每天随市场买卖波动——买的人多就涨，卖的人多就跌。

**本项目中的股票**：我们选择了 **28 只美国大盘股**（如 AAPL、MSFT、GOOGL 等），这些是纳斯达克和纽交所最大的上市公司，流动性好、数据容易获取。

**相关概念**：
- **Ticker / 股票代码**：每只股票的唯一缩写标识，如 AAPL = Apple, MSFT = Microsoft, GOOGL = Google
- **交易所**：股票买卖的场所，如 NYSE（纽约证券交易所）、NASDAQ（纳斯达克）
- **大盘股（Large Cap）**：市值很大的公司，通常更稳定，价格波动相对较小

---

## 1.2 OHLCV 数据

**大白话**：OHLCV 是股票每天的基本"档案"，包含 5 个数据：

| 缩写 | 英文 | 中文 | 含义 |
|------|------|------|------|
| **O** | Open | 开盘价 | 当天第一笔交易的价格 |
| **H** | High | 最高价 | 当天交易的最高价格 |
| **L** | Low | 最低价 | 当天交易的最低价格 |
| **C** | Close | 收盘价 | 当天最后一笔交易的价格（最重要的一个，我们主要用这个） |
| **V** | Volume | 成交量 | 当天总共买卖了多少股 |

**举个例子**：2026-05-15 的 AAPL OHLCV 可能是：
- Open: $189.50（开盘价 189.50 美元）
- High: $191.20（盘中最高到 191.20）
- Low: $189.00（盘中最低到 189.00）
- Close: $190.80（收盘价 190.80）
- Volume: 52,300,000（约 5230 万股）

**在代码中的位置**：`data_ingestion/market_data.py` 负责获取这些数据

## 1.3 技术指标

技术指标是通过对 OHLCV 数据进行数学计算得出的衍生数据，用来帮助判断股票的"趋势"和"超买超卖"状态。

### 1.3.1 移动平均线（Moving Average, MA）

**大白话**：把过去 N 天的收盘价取平均值，画成一条线。作用是**过滤掉每天的随机波动**，看清楚大趋势。

**常用类型**：
- **MA50**：过去 50 天的平均收盘价 → 反映短期趋势
- **MA200**：过去 200 天的平均收盘价 → 反映长期趋势

**判断规则**（简化版）：
- 收盘价 > MA50 → 短期趋势向上（偏多）
- 收盘价 < MA50 → 短期趋势向下（偏空）
- MA50 > MA200 → "金叉"，中长期看涨信号
- MA50 < MA200 → "死叉"，中长期看跌信号

**在项目中的使用**：
```python
# 状态向量中的特征：
MA50_ratio  = 当前价格 / MA50    # >1 说明在均线上方（偏多）
MA200_ratio = 当前价格 / MA200   # >1 说明在长期均线上方（偏多）
```

### 1.3.2 RSI（相对强弱指数, Relative Strength Index）

**大白话**：衡量股票最近一段时间"涨得快还是跌得快"。取值范围 0-100。

| RSI 值 | 含义 | 常见解读 |
|--------|------|----------|
| > 70 | 涨得太快了 | 可能"超买"（overbought），即将回调 |
| 30-70 | 正常波动 | 无明确信号 |
| < 30 | 跌得太快了 | 可能"超卖"（oversold），即将反弹 |

**计算方法**（不需要理解细节，知道含义即可）：
- 取过去 14 天的涨跌
- 计算平均涨幅和平均跌幅
- RSI = 100 × 平均涨幅 / (平均涨幅 + 平均跌幅)

**在项目中的使用**：
```python
RSI_norm = (RSI - 50) / 50  # 归一化到 [-1, 1]，0 = 中性
```

### 1.3.3 MACD（移动平均收敛发散指标）

**大白话**：比较两条不同速度的移动平均线之间的差距。当差距变大时说明趋势在加速，差距变小时说明趋势在减弱。

**组成**：
- **MACD 线**：快速 MA(12天) − 慢速 MA(26天) → 两条均线之间的距离
- **信号线**：MACD 线的 9 天移动平均
- **柱状图**：MACD 线 − 信号线 → 两条线的距离

**判断规则**：
- MACD > 0 且 MACD > 信号线 → 看涨（上涨动能增强）
- MACD < 0 且 MACD < 信号线 → 看跌（下跌动能增强）

**在项目中的使用**：
```python
MACD_ratio = MACD / |当前价格|  # 除以价格使不同价位股票可比
```

### 1.3.4 EMA（指数移动平均, Exponential Moving Average）

**大白话**：跟普通 MA 类似，但**给最近的数据更大的权重**。普通 MA 是每一天权重一样（1/50），而 EMA 是今天的数据权重更大，越早的数据权重越小。

**为什么用 EMA 而不是 MA**：EMA 对最新的变化反应更快。比如一条利好新闻今天出来，EMA 会比 MA 更快地反映价格变化。

**在项目中的使用**：`SENTIMENT_EMA_SPAN=2`，对情感信号做 2 天 EMA 平滑，减少日间噪声。

## 1.4 交易成本

**大白话**：买卖股票不是免费的。每次交易都要付手续费，就像在银行转账要付手续费一样。

**本项目设置**：
```python
TRANSACTION_COST_PCT = 0.001  # 每次交易收 0.1% 的手续费
```

**实际影响举例**：
- 你有 $100,000，买了一笔 $50,000 的股票
- 手续费 = $50,000 × 0.1% = $50
- 过几天卖了，又付 $50
- 来回一次交易成本 = $100
- 如果做了 19 笔交易 → 累计成本约 $1,900
- 这就是为什么模拟实盘 P&L ≈ −$99（接近零）——几乎所有微小的利润都被交易成本吃掉了

**为什么这个很重要**：交易成本是策略的"隐形杀手"。一个理论上能赚 1% 的策略，如果因为频繁交易付了 2% 的手续费，实际结果是亏 1%。

## 1.5 衡量策略好坏的指标

### 1.5.1 累计收益率（Cumulative Return）

**大白话**：从开始到结束，你总共赚了（或亏了）百分之多少。

```
累计收益率 = (期末资金 − 期初资金) / 期初资金 × 100%
```

**举例**：
- 期初投入 $100,000
- 期末变成 $102,000
- 累计收益率 = (102,000 − 100,000) / 100,000 = +2%

### 1.5.2 Sharpe Ratio（夏普比率）

**大白话**：衡量"每承受一份风险，你赚了多少回报"。**数字越大越好**。

```
Sharpe Ratio = (平均收益率 − 无风险利率) / 收益率的标准差
```

| Sharpe 值 | 评价 |
|-----------|------|
| < 0 | 亏钱，还不如把钱存银行 |
| 0 ~ 0.5 | 勉强及格，风险调整后收益很低 |
| 0.5 ~ 1.0 | 还不错 |
| 1.0 ~ 2.0 | 很好 |
| > 2.0 | 极好（但通常是过拟合的信号） |

**为什么用 Sharpe 而不是只看收益率**：因为高收益率可能只是因为运气好（承担了巨大风险），Sharpe 把"风险"也考虑进去了。赚 20% 但波动 50% 的策略（Sharpe ≈ 0.4）不如赚 10% 但波动只有 10% 的策略（Sharpe ≈ 1.0）。

**在项目中的角色**：Sharpe 是**最重要的评价指标**，消融实验用 ΔSharpe（Sharpe 的差值）来衡量 NLP 信号的价值。

### 1.5.3 Maximum Drawdown（最大回撤）

**大白话**：从最高点到最低点，你最多亏了多少。**衡量策略的"最坏情况"**。

```
最大回撤 = (最高点净值 − 最低点净值) / 最高点净值 × 100%
```

**举例**：
- 你的资金从 $100,000 涨到 $110,000（最高点）
- 然后跌到 $95,000（最低点）
- 最大回撤 = (110,000 − 95,000) / 110,000 = 13.6%

**为什么重要**：一个策略可能最终赚了 20%，但如果中间曾经亏了 50%，你可能早就忍不住割肉止损了。最大回撤衡量的是**心理承受能力的极限**。

**在仪表盘中的显示**：蓝色权益曲线 + 红色回撤区域（Portfolio Performance 面板）

### 1.5.4 年化波动率（Annualized Volatility）

**大白话**：收益率上下波动的剧烈程度。用收益率的标准差来衡量，再年化处理。

- 高波动率 = 每天涨跌幅很大 → 风险高
- 低波动率 = 每天涨跌幅很小 → 相对稳定

## 1.6 Buy-and-Hold 基准

**大白话**：最简单的策略——买入后一直持有，不操作。这是所有量化策略必须对比的"基准线"。

**为什么它是基准**：
1. 学术界大量研究表明，对大盘股来说，Buy-and-Hold 非常难被打败
2. 因为每一次主动交易都有交易成本，频繁交易反而可能亏钱
3. 如果你的复杂策略不能打败"什么都不做"，那这个策略就没有实际价值

**本项目的结果**：DQN+NLP 与 Buy-and-Hold 基本持平（Delta Sharpe ≈ 0.05~0.15，但 DQN 自身训练方差 ±0.3~0.5）。这不是"失败"，而是一个有价值的发现——见第八章消融实验。

# 第三章 数据采集层

## 3.1 市场数据采集

**负责文件**：`data_ingestion/market_data.py`

**功能**：获取 28 只美股的 OHLCV 数据（见 1.2 节），回溯 100 天的历史价格。

### 数据源回退链

**大白话**：我们准备了多个数据源，一个挂了自动换下一个，保证总能拿到数据。

```
尝试顺序：Yahoo Direct API → yfinance → Alpha Vantage → 合成数据（GBM）
```

| 优先级 | 数据源 | 说明 | 是否免费 |
|--------|--------|------|----------|
| 1️⃣ | Yahoo Direct API | 雅虎财经的直接接口，速度最快 | ✅ 免费，但有限流 |
| 2️⃣ | yfinance | Python 的雅虎财经封装库 | ✅ 免费 |
| 3️⃣ | Alpha Vantage | 专业金融数据 API | ✅ 免费（有每分钟 5 次限制） |
| 4️⃣ | 合成数据（GBM） | 自己用数学公式模拟生成 | ✅ 不需要网络 |

### 合成数据（GBM 几何布朗运动）

**大白话**：当所有 API 都失败了（比如网络断了、API key 过期），我们就用数学公式**模拟**生成股票价格数据。

**GBM 是什么**：金融学中模拟股票价格最经典的数学模型，基本思想是：
- 每天的价格变化 = **漂移项**（长期趋势）+ **随机项**（每天的随机波动）
- 模拟出来的价格走势跟真实股票看起来很像

**代码中每只股票有独立的参数**：
```python
# 每只股票有自己的"性格"：
{
    "base_price": 150.0,    # 基准价格（比如苹果约 150 美元起步）
    "volatility": 0.02,     # 波动率（每天涨跌幅约 2%）
    "drift": 0.0001         # 漂移率（长期微微上涨）
}
```

> ⚠️ 合成数据只是兜底方案。真实分析时用的都是前三个数据源的真实数据。

# 第四章 NLP 情感分析层

## 4.1 什么是情感分析

**大白话**：让电脑自动判断一段文字是"正面"（看好）、"负面"（看空）、还是"中性"的。

**举例**：
- "Apple reports record-breaking Q4 revenue" → **正面**（利好，应该买）
- "Tesla faces major recall over safety concerns" → **负面**（利空，应该卖）
- "Microsoft announces new CEO transition plan" → **中性**（信息不足，不操作）

**输出格式**：每条新闻给出一个 **情感分数（sentiment score）**：
- 正数 = 正面情感（看好），数值越大越正面
- 负数 = 负面情感（看空），数值越小越负面
- 0 = 中性

**为什么要做情感分析**：因为股票价格会受新闻影响。如果能自动、快速地分析大量新闻的情感，就能比人更快地做出交易决策。

## 4.2 方法一：VADER

**负责文件**：`nlp_pipeline/sentiment_vader.py`

### VADER 是什么

**大白话**：VADER 是一个基于"词典+规则"的情感分析工具。它里面有一个巨大的词典，每个词都有一个情感分数。

**工作原理**：
1. 把新闻文本拆成单词
2. 查词典找到每个词的情感分数
3. 应用一些规则修正最终得分

**举例**：
```
"Apple's profits GREATLY exceeded expectations 😃"

分词 → ["Apple's", "profits", "GREATLY", "exceeded", "expectations", "😃"]

词典查找 →
  "profits"    : +0.3（偏正面）
  "GREATLY"    : +0.5（大写强调 → 分数放大）
  "exceeded"   : +0.2（偏正面）
  "expectations": +0.1（偏正面）
  "😃" (emoji) : +0.9（笑脸 → 很正面）

最终得分 ≈ +0.7（正面情感）
```

### VADER 的规则修正

| 规则 | 举例 | 效果 |
|------|------|------|
| 大写强调 | "REALLY bad" vs "really bad" | 加强程度 |
| 否定词 | "not good" → 翻转正面词 | "好"变"不好" |
| 程度副词 | "extremely good" | 放大正面分数 |
| 情感词至少 3 个才计分 | "The book is good"（只有 1 个）→ 降低整体强度 | 防止短文本误判 |

### VADER 的优缺点

| 优点 | 缺点 |
|------|------|
| ⚡ 速度极快（不需要训练，直接用） | ❌ 不理解上下文，只能看单个词 |
| 📦 不需要 GPU | ❌ 金融术语可能不在词典中 |
| 🎯 处理社交媒体文本效果好 | ❌ 无法理解复杂句子的逻辑关系 |

---

## 4.3 方法二：Logistic Regression（逻辑回归）

**负责文件**：`nlp_pipeline/sentiment_lr.py`

### 逻辑回归是什么

**大白话**：一种简单的机器学习分类器。给它看很多"标注好情感"的文本，它学会一个模式，然后用这个模式来判断新文本的情感。

**跟 VADER 的区别**：VADER 是靠"查词典"，逻辑回归是靠"学过的经验"。

### 训练过程

**问题**：我们没有人工标注的数据（没人给 3 万条新闻逐条标注"正面/负面/中性"）。

**解决**：用**合成标签（synthetic labels）**——基于关键词自动生成训练标签。

```python
# 合成标签示例：
正面关键词列表 = ["beat", "surge", "profit", "growth", "record", "upgrade", ...]
负面关键词列表 = ["miss", "drop", "loss", "decline", "cut", "downgrade", ...]

# 如果新闻包含 2 个以上正面关键词且无负面关键词 → 标为正面
# 如果新闻包含 2 个以上负面关键词且无正面关键词 → 标为负面
# 否则 → 标为中性
```

### TF-IDF 特征

**大白话**：把文本转换成数字的方法。核心思想是：**某些词出现频率高但所有文章都有（如"the"、"is"）→ 不重要；某些词只在某些文章中出现 → 很重要**。

| 缩写 | 英文 | 中文 |
|------|------|------|
| TF | Term Frequency | 词频：一个词在这篇文章中出现多少次 |
| IDF | Inverse Document Frequency | 逆文档频率：这个词在所有文章中有多"罕见" |
| TF-IDF | - | TF × IDF：词频 × 罕见度 → 越大说明这个词在这篇文章中越"有代表性" |

**举例**：
```
文章："Apple revenue surged on strong iPhone sales"

"surged" 的 TF-IDF 高（在金融新闻中罕见且出现）
"the" 的 TF-IDF 低（所有文章都有）
```

### 逻辑回归的优缺点

| 优点 | 缺点 |
|------|------|
| 🏃 训练和推理都快 | ❌ 合成标签有噪声（可能标错） |
| 📊 可解释（能看到哪些词权重最高） | ❌ 无法理解上下文和语义 |
| 💾 模型文件很小 | ❌ 对没见过的词组/表达处理不好 |

---

## 4.4 方法三：FinBERT

**负责文件**：`nlp_pipeline/sentiment_finbert.py`

### FinBERT 是什么

**大白话**：FinBERT 是一个用**深度学习（Transformer）** 训练的情感分析模型。它不仅能看单个词，还能**理解整句话的语义和上下文**。

**跟 VADER 和 LR 的本质区别**：

| | VADER | LR | FinBERT |
|--|-------|-----|---------|
| 理解单个词 | ✅ | ✅ | ✅ |
| 理解词的搭配 | ❌ | ✅ | ✅ |
| 理解句子语义 | ❌ | ❌ | ✅ |
| 理解上下文 | ❌ | ❌ | ✅ |

**举例展示 FinBERT 的优势**：
```
"The company exceeded expectations, BUT margins are shrinking."

VADER:  可能判为正面（因为 "exceeded" 很正面）
LR:     可能判为中性（正负关键词各一个，互相抵消）
FinBERT: 判为负面（理解了 BUT 的转折关系，重点在后半句）
```

### 技术细节

- **模型名称**：`ProsusAI/finbert`（由 Prosus 公司在 HuggingFace 上开源）
- **预训练**：先用大量通用文本训练 BERT，再用金融新闻数据微调（fine-tune）
- **推理设备**：自动选择 GPU → MPS(Apple Silicon) → CPU

### FinBERT 的优缺点

| 优点 | 缺点 |
|------|------|
| 🧠 理解力最强（能处理复杂语义） | 🐢 推理速度最慢 |
| 🎯 金融领域效果最好 | 💻 需要 GPU（或 Apple Silicon）才能快速运行 |
| 📊 输出概率分布（正面/负面/中性各多少概率） | 📦 模型文件很大（约 420MB） |

---

## 4.5 三种方法的对比

| 维度 | VADER | Logistic Regression | FinBERT |
|------|-------|---------------------|---------|
| **技术路线** | 词典 + 规则 | 机器学习 + TF-IDF | 深度学习（Transformer） |
| **速度** | ⚡⚡⚡ 极快 | ⚡⚡ 快 | ⚡ 较慢 |
| **准确度** | ⭐⭐ 一般 | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐ 最高 |
| **上下文理解** | ❌ | ❌ | ✅ |
| **需要训练** | ❌ 不需要 | ✅ 需要（合成标签） | ❌ 不需要（已预训练） |
| **资源需求** | 无 | CPU 即可 | GPU/MPS 推荐 |
| **输出** | 复合分数 [-1, 1] | 类别概率 | 类别概率 |

**为什么要用三种方法**：如果只依赖一种方法，万一它判错了怎么办？用三种方法互相"投票"，结果更可靠。就像法院判决不能只听一个法官的——见下一节"共识机制"。

---

## 4.6 共识机制（Consensus Mechanism）

**负责文件**：`nlp_pipeline/aggregator.py` 中的 `compute_consensus_score()`

### 为什么需要共识

**大白话**：三种方法对同一条新闻可能给出不同的判断。共识机制就是"投票表决"——综合三种方法的结果，得出一个最终的、更可靠的判断。

**举例**：
```
新闻："Apple Q3 revenue up 15%, beating analyst estimates"

VADER:   +0.8（正面，看到 "up", "beating"）
LR:      +0.5（正面，关键词匹配）
FinBERT: +0.9（正面，理解语义）

共识 → 三者一致正面 → 最终分数高，可以高信心买入
```

```
新闻："Apple launches new product amid supply chain concerns"

VADER:   +0.3（正面，看到 "launches", "new"）
LR:      -0.1（负面，"concerns" 权重高）
FinBERT: +0.1（正面偏中性，理解了 "amid" 的转折）

共识 → 三者不一致 → 最终分数接近 0，不操作
```

### 加权投票规则

**大白话**：不是简单的"少数服从多数"，而是给更靠谱的方法更大的权重。

```python
# 默认权重：
FinBERT 权重 = 2.0  （最准确，权重最大）
LR 权重     = 1.0
VADER 权重  = 1.0

# 特殊情况：当三种方法一致性低时（Fleiss' Kappa < 0.6）
# 说明大家对新闻的解读分歧很大
# 此时提升 LR 的权重来增加多样性：
LR 权重 = 1.5  （从 1.0 提升到 1.5）
```

### 计算流程（一步步来）

```
第 1 步：逐篇打分
  每条新闻 → VADER 打分 + LR 打分 + FinBERT 打分

第 2 步：日级别聚合
  把同一天、同一只股票的多条新闻分数，按置信度加权平均
  → 得到每天的 sentiment_score

第 3 步：多方法共识
  对三种方法的日级别分数，按权重加权平均
  → 得到共识分数

第 4 步：EMA 平滑（见 4.7 节）
  减少日间噪声

第 5 步：中性门控（见 4.8 节）
  过滤掉弱信号
```

---

## 4.7 EMA 平滑

**大白话**：情感分数每天都有波动（今天 +0.3，明天 -0.1，后天 +0.2），这些随机波动是"噪声"。EMA 平滑就像"擦玻璃"——让曲线变得更平滑，更容易看出趋势。

**EMA vs 普通平均**：
```
普通平均（MA）：每天的权重一样
  3 天平均 = (今天 + 昨天 + 前天) / 3

EMA：今天的权重最大，昨天次之，前天最小
  EMA(今天) = 0.67 × 今天 + 0.22 × 昨天 + 0.11 × 前天
```

**为什么选 EMA**：因为我们要让模型对**最新的情感变化反应更快**。如果今天突然出了一条重磅利好新闻，EMA 会比普通平均更快地反映这个变化。

**项目中的设置**：
```python
SENTIMENT_EMA_SPAN = 2  # 2 天 EMA，反应很快
```

---

## 4.8 中性门控（Neutral Gating）

**大白话**：有些新闻的情感很弱（比如 "Apple held a meeting today"），这种新闻对股价基本没影响。中性门控就是**把情感太弱的信号强制归零**，避免模型被无用信号误导。

```python
# 简化逻辑：
if abs(sentiment_score) < threshold:
    sentiment_score = 0  # 太弱了，直接归零
```

**为什么要做这个**：如果不做门控，大量中性但非零的弱信号会"稀释"真正有用的强信号，让模型分不清什么是重要信息、什么是噪音。

---

## 4.9 一致性评估指标

**负责文件**：`nlp_pipeline/aggregator.py`

项目用两个指标来衡量三种方法"意见是否一致"：

### 4.9.1 Fleiss' Kappa（弗莱斯 Kappa 系数）

**大白话**：衡量多个"评委"之间的一致程度。取值范围 0~1。

| Kappa 值 | 含义 |
|----------|------|
| 0 ~ 0.2 | 几乎没有一致性（各说各的） |
| 0.2 ~ 0.4 | 轻微一致 |
| 0.4 ~ 0.6 | 中等一致 |
| 0.6 ~ 0.8 | 高度一致 |
| 0.8 ~ 1.0 | 几乎完全一致 |

**在项目中的应用**：
```python
if kappa < 0.6:
    # 三种方法分歧大 → 提升 LR 权重增加多样性
    lr_weight = 1.5
else:
    # 三种方法一致 → 维持默认权重
    lr_weight = 1.0
```

### 4.9.2 逐对 Pearson 相关系数

**大白话**：每两种方法之间算一个相关系数，看它们的打分趋势是否一致。

```
VADER vs LR：      r = 0.65 （中等正相关）
VADER vs FinBERT：  r = 0.72 （较强正相关）
LR vs FinBERT：     r = 0.58 （中等正相关）
```

> 相关系数越接近 1，说明两种方法的判断越一致。

### 4.9.3 F1 vs FinBERT

**大白话**：以 FinBERT 为"标准答案"（伪基准），计算 VADER 和 LR 的 F1 分数。

**F1 分数**：精确率（Precision）和召回率（Recall）的调和平均数。取值 0~1，越高越好。

- **精确率**：VADER 说"正面"的新闻中，有多少 FinBERT 也认为是"正面"
- **召回率**：FinBERT 认为"正面"的新闻中，有多少 VADER 也识别出来了

# 第五章 强化学习基础

## 5.1 什么是强化学习（RL）

**大白话**：强化学习就是**通过试错来学习**。就像训练小狗：
1. 你给小狗一个指令（状态）
2. 小狗做一个动作
3. 如果动作对了，给零食（正奖励）；如果错了，不给（负奖励/零奖励）
4. 小狗会逐渐学会"在什么情况下做什么动作能获得最多奖励"

**RL 的核心要素**：

| 要素 | 含义 | 本项目对应 |
|------|------|-----------|
| **Agent（智能体）** | 学习和做决策的"AI" | DQN 网络 |
| **Environment（环境）** | Agent 所处的世界 | 股票市场（Gym 环境） |
| **State（状态）** | Agent 看到的信息 | 11 维特征向量（价格、指标、情感等） |
| **Action（动作）** | Agent 能做的操作 | Buy / Sell / Hold |
| **Reward（奖励）** | 动作之后的反馈 | 盈亏 + 多种惩罚/奖励组件 |

**跟监督学习的区别**：
- **监督学习**（如 LR）：有"标准答案"，学习模仿
- **强化学习**：没有标准答案，通过试错自己摸索最优策略

---

## 5.2 Q-Learning 核心思想

**大白话**：Q-Learning 的核心是一个"Q 表"——记录**在每个状态下、每个动作的"期望价值"**。

### Q 表举例

假设只有 2 个状态（涨/跌）和 3 个动作（买/卖/持有）：

| 状态 | Buy | Sell | Hold |
|------|-----|------|------|
| 涨 | +10 （赚钱！） | -5 （卖了亏了） | +3 （小赚） |
| 跌 | -8 （买了亏了） | +6 （成功逃顶） | -1 （小亏） |

**Agent 的策略就是**：在当前状态下，选 Q 值最大的动作。
- 涨的时候 → 选 Buy（Q=10 最大）
- 跌的时候 → 选 Sell（Q=6 最大）

### Q 值更新公式（贝尔曼方程）

```
Q(状态, 动作) = Q(状态, 动作) 
  + 学习率 × [ 立即奖励 + 折扣因子 × max(Q(下一状态)) − Q(状态, 动作) ]
```

**大白话**：每次做完一个动作，根据"实际得到的奖励"来更新 Q 表中的值。如果实际奖励比预期好，就提高 Q 值；如果比预期差，就降低 Q 值。

**Q-Learning 的问题**：当状态很多时（本项目有 11 维连续状态，理论上无限多种组合），Q 表根本存不下。**解决方案：用神经网络来近似 Q 函数 → DQN。**

---

## 5.3 从 Q-Learning 到 DQN

**大白话**：DQN = Deep Q-Network，就是**用深度神经网络代替 Q 表**。

**核心思想**：
- Q-Learning：用表格存 Q 值 → 只适合状态空间小的问题
- DQN：用神经网络计算 Q 值 → 可以处理高维连续状态

**工作流程**：
```
输入：11 维状态向量（价格、指标、情感...）
  ↓
DQN 神经网络（4 层全连接）
  ↓
输出：3 个 Q 值（Buy 的 Q值, Sell 的 Q值, Hold 的 Q值）
  ↓
选 Q 值最大的动作执行
```

---

## 5.4 Double DQN（本项目使用）

**大白话**：普通 DQN 有一个"过于乐观"的毛病——它总是高估 Q 值。Double DQN 的修正是：用两个网络，一个选动作，一个评估动作，避免过度乐观。

**对比**：

| | 普通 DQN | Double DQN |
|--|----------|------------|
| 选动作 | 网络自己选 | 在线网络选 |
| 评估动作 | 同一个网络评估 | 用目标网络评估 |
| 问题 | 容易高估 Q 值 | 更保守，Q 值估计更准确 |

**本项目实现**：`rl_engine/dqn.py` 中实现了 Double DQN 的更新逻辑。

# 第六章 RL 交易环境

**负责文件**：`rl_engine/env.py` — `FinancialTradingEnv`

## 6.1 交易环境是什么

**大白话**：交易环境就是一个**模拟的股票市场**。DQN Agent 在这个环境里"做交易"——看到市场状态 → 做出买卖决定 → 获得盈亏反馈 → 学习改进。

**每一步的流程**：
```
1. 环境提供当前状态（State）→ 11 维特征向量
2. Agent 根据状态选择动作（Action）→ Buy / Sell / Hold
3. 环境执行交易，计算盈亏
4. 环境返回新的状态 + 奖励（Reward）
5. 重复以上步骤
```

**环境参数**（来自 `config.py`）：

```python
INITIAL_CAPITAL = 100_000        # 初始资金 $100,000
TRANSACTION_COST_PCT = 0.001     # 交易成本 0.1%
LOOKBACK_WINDOW = 100            # 回看 100 天数据
```

## 6.2 状态向量（State Vector）— 11 个特征逐一解释

**大白话**：状态向量就是 Agent 每一步"看到的世界"。就像你做交易决策时需要看的各种信息——价格是多少、趋势怎样、新闻好不好、当前持仓多少……

### 技术指标特征（7 个，来自 OHLCV 数据）

| # | 特征名 | 计算方式 | 含义 | 范围 |
|---|--------|----------|------|------|
| 1 | `price_ratio` | 当前价格 / lookback窗口均价 | 当前价格处于相对高位还是低位 | ≈ 0.9~1.1 |
| 2 | `MA50_ratio` | 当前价格 / 50日均线 | 短期趋势方向 | ≈ 0.8~1.2 |
| 3 | `MA200_ratio` | 当前价格 / 200日均线 | 长期趋势方向 | ≈ 0.7~1.3 |
| 4 | `RSI_norm` | (RSI − 50) / 50 | 超买/超卖程度 | −1 ~ +1 |
| 5 | `MACD_ratio` | MACD值 / |当前价格| | 动能方向和强度 | ≈ −0.05~+0.05 |
| 6 | `position_pct` | 当前持仓市值 / 总资金 | 有多少资金在股市里 | 0 ~ 1 |
| 7 | `cash_pct` | 现金 / 总资金 | 手里还有多少现金 | 0 ~ 1 |

**详细解释**：

**#1 price_ratio — 价格水平**
```
lookback均价 = 过去100天收盘价的平均值
price_ratio = 今天的价格 / lookback均价

例：均价 $180，今天 $190 → ratio = 1.056（在均价上方，偏多）
例：均价 $180，今天 $170 → ratio = 0.944（在均价下方，偏空）
```

**#2-3 MA50_ratio / MA200_ratio — 趋势方向**
```
MA50_ratio > 1 → 价格在短期均线上方 → 短期看涨
MA50_ratio < 1 → 价格在短期均线下方 → 短期看跌
MA200_ratio > 1 → 价格在长期均线上方 → 长期看涨
两者结合：
  MA50 > 1 且 MA200 > 1 → 多头趋势（涨势确认）
  MA50 < 1 且 MA200 < 1 → 空头趋势（跌势确认）
  MA50 > 1 但 MA200 < 1 → 可能是短期反弹
```

**#4 RSI_norm — 超买超卖**
```
RSI_norm = (RSI − 50) / 50

RSI = 70 → norm = (70-50)/50 = +0.4（偏超买）
RSI = 50 → norm = 0（中性）
RSI = 30 → norm = (30-50)/50 = -0.4（偏超卖）
```

**#5 MACD_ratio — 动能方向**
```
MACD_ratio = MACD值 / |当前价格|

> 0 → MACD 在零轴上方 → 上涨动能
< 0 → MACD 在零轴下方 → 下跌动能
```

**#6-7 position_pct / cash_pct — 资金分布**
```
position_pct = 0.7, cash_pct = 0.3 → 70% 在股市，30% 现金
position_pct = 1.0, cash_pct = 0.0 → 全仓（所有钱都在股市里）
position_pct = 0.0, cash_pct = 1.0 → 空仓（所有钱都是现金）

两者之和应该 = 1（或接近 1，因为交易成本会吃掉一点）
```

### 情感指标特征（4 个，来自 NLP 分析）

| # | 特征名 | 计算方式 | 含义 | 范围 |
|---|--------|----------|------|------|
| 8 | `sentiment_score` | 当日共识情感分数 | 今天的新闻情绪 | −1 ~ +1 |
| 9 | `sentiment_ma5` | 情感分数的 5 日均值 | 近 5 天的平均情绪 | −1 ~ +1 |
| 10 | `sentiment_trend` | 情感分数的 5 日差分 | 情绪在变好还是变差 | ≈ −2~+2 |
| 11 | `sentiment_vol` | 情感分数的 5 日标准差 | 情绪的波动程度 | 0 ~ +1 |

**详细解释**：

**#8 sentiment_score — 当日情绪**
```
+0.8 → 今天新闻整体很正面 → 看好这只股票
-0.5 → 今天新闻整体偏负面 → 看空这只股票
 0.0 → 中性或被门控过滤
```

**#9 sentiment_ma5 — 近期平均情绪**
```
跟 price_ratio 类似的思路：
sentiment_ma5 > 0 → 近 5 天新闻偏正面 → 中短期情绪看涨
sentiment_ma5 < 0 → 近 5 天新闻偏负面 → 中短期情绪看跌
```

**#10 sentiment_trend — 情绪趋势**
```
sentiment_trend = 今天的 sentiment_score − 5天前的 sentiment_score

> 0 → 情绪在改善（从负面转向正面，或正面更强）
< 0 → 情绪在恶化（从正面转向负面，或负面更强）

例：5天前 -0.3，今天 +0.2 → trend = +0.5（情绪明显好转）
例：5天前 +0.6，今天 +0.1 → trend = -0.5（情绪在衰退）
```

**#11 sentiment_vol — 情绪波动**
```
高波动（接近1）→ 新闻时好时坏，市场不确定
低波动（接近0）→ 新闻情绪稳定一致

高波动时的策略意义：Agent 可能应该更谨慎（减少交易）
低波动时的策略意义：情绪信号更可靠
```

### 状态向量完整示例

```
AAPL 在 2026-05-15 的状态向量：
┌─────────────────────┬────────┬──────────────────────┐
│ 特征                 │ 值     │ 解读                  │
├─────────────────────┼────────┼──────────────────────┤
│ price_ratio         │  1.03  │ 价格在均价上方 3%     │
│ MA50_ratio          │  1.01  │ 在短期均线上方（偏多）  │
│ MA200_ratio         │  1.08  │ 在长期均线上方（多头）  │
│ RSI_norm            │  0.15  │ RSI ≈ 57.5（中性偏多）│
│ MACD_ratio          │  0.002 │ MACD 微正（弱上涨动能） │
│ position_pct        │  0.6   │ 60% 资金在股市         │
│ cash_pct            │  0.4   │ 40% 现金               │
│ sentiment_score     │  0.5   │ 今日新闻偏正面          │
│ sentiment_ma5       │  0.3   │ 近5天平均偏正面         │
│ sentiment_trend     │  0.2   │ 情绪在改善              │
│ sentiment_vol       │  0.15  │ 情绪波动不大            │
└─────────────────────┴────────┴──────────────────────┘
```

## 6.3 动作空间（Action Space）

**大白话**：Agent 每一步只能做 3 件事中的一个。

| 动作 | 编号 | 含义 | 对应操作 |
|------|------|------|----------|
| **Hold** | 0 | 不操作 | 什么都不做，维持当前持仓 |
| **Buy** | 1 | 买入 | 用现金买入股票（买总资金的 10%） |
| **Sell** | 2 | 卖出 | 卖出部分股票（卖持仓的 10%） |

**为什么只有 3 个动作**：这是**离散动作空间**（Discrete Action Space），比连续动作（如"买入 15.7% 的资金"）更容易训练且更稳定。

**为什么每次买/卖 10%**：这是代码中固定的交易比例，避免一次全部买入（风险太大）或一次全卖（可能错过后续上涨）。

## 6.4 奖励函数（Reward Function）— 5 个组件逐一解释

**大白话**：奖励函数就是"评分标准"——Agent 做完一个动作后，环境根据多个维度给一个综合分数。Agent 的目标就是最大化这个综合分数。

### 完整公式

```
总奖励 = 收益奖励 + 现金惩罚 + 集中度惩罚 + 交易频率惩罚 + 情感对齐奖励
```

### 组件 1：收益奖励（pct_return）

**大白话**：最核心的奖励——你的持仓今天赚了还是亏了。

```
pct_return = (当前总资产 − 上一步总资产) / 上一步总资产
```

| 情况 | pct_return | 含义 |
|------|-----------|------|
| 赚了 1% | +0.01 | 正奖励 ✅ |
| 亏了 0.5% | -0.005 | 负奖励 ❌ |
| 没变化 | 0.0 | 零奖励 |

### 组件 2：现金惩罚（cash_penalty）

**大白话**：如果你有大量现金却什么都不做，就扣分。这是在**鼓励 Agent 积极利用资金**。

```python
# 简化逻辑：
if cash_pct > 0.5:  # 超过一半资金是现金
    cash_penalty = -0.001 × (cash_pct - 0.5)
else:
    cash_penalty = 0
```

**为什么要惩罚**：如果 Agent 学会了"永远持有现金不动"，那跟把钱存银行没区别。这个惩罚逼着 Agent 去找投资机会。

### 组件 3：集中度惩罚（concentration_penalty）

**大白话**：如果你把几乎所有钱都押在一只股票上，就扣分。这是在**鼓励分散投资、降低风险**。

```python
# 简化逻辑：
if position_pct > 0.9:  # 超过 90% 的资金在一只股票上
    concentration_penalty = -0.002 × (position_pct - 0.9)
else:
    concentration_penalty = 0
```

**为什么要惩罚**：集中投资风险太大——万一这只股票暴跌，你就血本无归。

### 组件 4：交易频率惩罚（frequency_penalty）

**大白话**：如果你买卖太频繁，就扣分。这是在**减少不必要的交易成本**。

```python
# 每次非 Hold 动作都会触发：
frequency_penalty = -0.0001  # 固定的小额惩罚
```

**为什么要惩罚**：还记得 1.4 节说的交易成本吗？每次交易要付 0.1% 的手续费。如果 Agent 频繁买卖，手续费会吃掉所有利润。这个惩罚让 Agent 学会"只在有把握时才交易"。

### 组件 5：情感对齐奖励（sentiment_bonus）

**大白话**：如果你的持仓方向跟新闻情感一致（正面新闻时持仓、负面新闻时空仓），就加分。这是在**鼓励 Agent 听新闻的**。

```python
# 简化逻辑：
if sentiment > 0 and position > 0:    # 正面新闻 + 有持仓
    sentiment_bonus = +0.001          # 加分（你做对了）
elif sentiment < 0 and position == 0: # 负面新闻 + 空仓
    sentiment_bonus = +0.001          # 加分（成功避险）
else:
    sentiment_bonus = 0               # 不加分
```

**为什么要奖励**：这是整个项目"用 NLP 帮助交易"的核心机制。这个奖励引导 Agent 学会"听新闻做决策"——当新闻好时买入，新闻差时卖出或空仓。

### 奖励函数完整示例

```
场景：AAPL 今天涨了 1%，新闻正面，Agent 持仓 60%

收益奖励:      +0.010  （持仓涨了 1%）
现金惩罚:      -0.0005 （40% 现金，超过 50% 的一半，但 40% < 50%，所以 0）
集中度惩罚:     0.0000  （60% 持仓 < 90%，不触发）
交易频率惩罚:    0.0000  （今天执行的是 Hold，不触发）
情感对齐奖励:  +0.0010  （正面新闻 + 有持仓，对齐）

总奖励 = +0.010 + 0 + 0 + 0 + 0.001 = +0.011 ✅
```

```
场景：AAPL 今天跌了 0.5%，新闻负面，Agent 全仓

收益奖励:      -0.005  （持仓跌了 0.5%）
现金惩罚:       0.0000 （0% 现金）
集中度惩罚:    -0.0020 （100% 持仓 > 90%，触发惩罚）
交易频率惩罚:   -0.0001 （之前做了 Sell 操作）
情感对齐奖励:   0.0000  （负面新闻但有持仓，不对齐）

总奖励 = -0.005 + 0 - 0.002 - 0.0001 + 0 = -0.0071 ❌
```

## 6.5 Walk-Forward 验证

**大白话**：按时间顺序把数据切成三段——用过去的数据训练，用中间的数据调优，用未来的数据测试。**严格按时间先后，绝不"偷看未来"**。

```
时间线：
├────── 训练集 ──────┤── 验证集 ──┤── 测试集 ──┤
├────── 60% ────────┤──── 20% ───┤──── 20% ───┤
      200天前              67天前        现在
```

| 数据集 | 时间范围 | 用途 |
|--------|----------|------|
| 训练集 | 最早 60% | Agent 学习交易策略 |
| 验证集 | 中间 20% | 调整超参数，防止过拟合 |
| 测试集 | 最近 20% | 最终评估（Agent 从未见过这些数据） |

**为什么不能打乱顺序**：如果打乱顺序，训练时可能"偷看"到了未来数据（比如用 5 月的数据预测 3 月的价格），这在真实交易中是不可能的。Walk-Forward 严格保证"只用过去预测未来"。

# 第七章 DQN 网络架构

**负责文件**：`rl_engine/dqn.py`

## 7.1 网络结构

**大白话**：一个 4 层的"数字大脑"——输入 11 个数字，经过 3 层内部计算，输出 3 个数字。

```
输入层        隐藏层1       隐藏层2       隐藏层3       输出层
11个特征  →  256个神经元  →  128个神经元  →  64个神经元  →  3个Q值
(price,     (ReLU)        (ReLU)        (ReLU)       (Buy的Q值,
 MA, RSI,                                             Sell的Q值,
 sentiment...)                                         Hold的Q值)
```

### 各层解释

| 层 | 神经元数 | 激活函数 | 作用 |
|----|---------|---------|------|
| 输入层 | 11 | — | 接收状态向量 |
| 隐藏层1 | 256 | ReLU | 第一层特征提取 |
| 隐藏层2 | 128 | ReLU | 第二层特征组合 |
| 隐藏层3 | 64 | ReLU | 第三层特征压缩 |
| 输出层 | 3 | — | 输出 Buy/Sell/Hold 的 Q 值 |

### 什么是 ReLU

**大白话**：ReLU 就是一个"过滤器"——把负数变成 0，正数不变。

```
输入:  -3   -1   0   2   5
ReLU:  0    0   0   2   5
```

**为什么用 ReLU**：简单、计算快、效果好。它让网络能学习"非线性"的关系（比如"价格涨到一定程度后反而该卖"）。

### 为什么网络越来越小（256 → 128 → 64）

**大白话**：像漏斗一样，逐步把信息"浓缩"。输入 11 个特征的信息量不大，不需要一个巨大的网络。逐步缩小可以：
1. 减少计算量（训练更快）
2. 防止过拟合（参数少 = 不容易"死记硬背"）
3. 强迫网络学习最关键的模式

## 7.2 经验回放缓冲（Experience Replay）

**大白话**：Agent 每一步的经历（状态、动作、奖励、新状态）都存进一个"记忆库"。训练时不是只用最新的经历，而是**从记忆库中随机抽取一批**来学习。

### 为什么需要经验回放

**问题 1：相关性**
- 连续两步的数据高度相关（价格变化是连续的）
- 如果只用连续数据训练，网络会"偏科"

**问题 2：遗忘**
- 早期学到的重要经验（如"大跌时该卖"）可能被后来的数据淹没

**经验回放的解决方案**：
```
经历1: (状态A, Buy, +0.01, 状态B)  ←───┐
经历2: (状态B, Hold, 0.00, 状态C)  ←───┤ 全部存进记忆库
经历3: (状态C, Sell, -0.02, 状态D)  ←───┤ (经验回放缓冲)
经历4: (状态D, Buy, +0.03, 状态E)  ←───┘

训练时，随机抽取一批（比如经历1和经历4）→ 打破时间顺序 → 更好的学习效果
```

## 7.3 目标网络（Target Network）

**大白话**：DQN 实际上有**两个一模一样的网络**：
- **在线网络（Online Network）**：每一步都在更新，用来做决策
- **目标网络（Target Network）**：每隔一段时间才同步一次，用来计算"目标 Q 值"

### 为什么需要目标网络

**问题**：Q-Learning 的更新公式中，"目标 Q 值"本身也在变化（因为网络参数每步都在变）。这就像"追逐一个移动的目标"——目标一直在动，很难收敛。

**解决方案**：用目标网络"冻结"目标值一段时间，让在线网络有稳定的"追击目标"。

```python
TARGET_UPDATE_FREQ = 5  # 每 5 步同步一次
```

**比喻**：
- 在线网络 = 学生（每天在学习新知识）
- 目标网络 = 老师出的模拟题（每 5 天才换一批新题）
- 学生对照固定的模拟题来检验自己的学习效果，而不是每天追着变化的题目跑

## 7.4 ε-greedy 探索策略

**大白话**：Agent 不能总选"当前认为最好"的动作——偶尔也要"瞎试一下"，否则可能错过更好的策略。

### 工作机制

```
每一步，Agent 以概率 ε 选择随机动作（探索），以概率 (1-ε) 选择最优动作（利用）

ε = 1.0  → 100% 随机（刚开始，什么都不懂，全靠试）
ε = 0.5  → 50% 随机，50% 选最优
ε = 0.1  → 10% 随机，90% 选最优（后期，已经学了很多，主要靠经验）
ε = 0.01 → 1% 随机，99% 选最优（最终，几乎不探索了）
```

### ε 衰减

```python
# 每个 episode 结束后，ε 减少：
ε = ε × decay_rate

# 从 1.0 开始，逐步衰减到 0.01
# 就像小孩长大：小时候什么都想试（高 ε），长大了知道该怎么做（低 ε）
```

**为什么要探索**：如果 Agent 从不探索，可能永远发现不了更好的策略。比如它一直 Hold，就永远不知道 Buy 其实能赚更多。

## 7.5 学习率调度与梯度裁剪

### 学习率调度（ExponentialLR）

**大白话**：学习率就是 Agent 每次"更新知识"的幅度。
- 学习率太大 → 更新太猛，可能"矫枉过正"
- 学习率太小 → 学习太慢，可能永远学不会

**ExponentialLR 衰减**：训练开始时学习率高（学得快），训练后期学习率低（学得精细）。

```
学习率变化：0.001 → 0.0008 → 0.0006 → 0.0005 → ...（指数衰减）
```

**比喻**：学自行车——开始时大幅调整方向（学习率高），熟练后做微调（学习率低）。

### 梯度裁剪（Gradient Clipping）

**大白话**：如果某次更新方向特别极端（比如"明天全仓买入"的冲动），就把更新幅度"剪"到一个合理范围内。

```python
# 防止某次参数更新太大导致训练崩溃
torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
```

**为什么需要**：偶尔训练数据中会有极端值（如股票单日涨跌 20%），导致梯度异常大。如果不裁剪，可能让网络参数"跳飞"，之前学到的东西全毁了。

# 第八章 消融实验

## 8.1 什么是消融实验

**大白话**：消融实验就是"拆零件测试"——一个系统有很多组件，**逐个关掉某个组件，看系统效果变化了多少**。变化越大说明这个组件越重要。

**生活中的比喻**：
```
你做了一碗汤，里面有：盐、味精、酱油、醋、辣椒

消融实验就是：
1. 正常汤 → 味道 9 分
2. 不加盐 → 味道 4 分 → 盐贡献了 5 分（非常重要！）
3. 不加味精 → 味道 8 分 → 味精贡献了 1 分（有点用但不是关键）
4. 不加醋 → 味道 9 分 → 醋贡献了 0 分（可有可无）
```

**本项目中的消融**：
```
完整系统 = 技术指标（价格、MA、RSI、MACD）+ NLP 情感信号
消融 = 关掉 NLP → 只用技术指标

对比两者的表现 → NLP 信号到底有没有用
```

## 8.2 本项目的消融设计

**负责文件**：`rl_engine/evaluation.py` 中的 `run_ablation_study()`

### 两种配置

| 配置 | 状态向量维度 | 包含的情感特征 | 用途 |
|------|-------------|---------------|------|
| **NLP-ON** | 11 维 | ✅ 包含 4 维情感特征 | 完整系统 |
| **NLP-OFF** | 7 维 | ❌ 去掉 4 维情感特征 | 消融对照 |

```
NLP-ON 状态：[price, MA50, MA200, RSI, MACD, pos, cash, sentiment, sent_ma5, sent_trend, sent_vol]
NLP-OFF 状态：[price, MA50, MA200, RSI, MACD, pos, cash]
                                                    ↑↑↑↑↑↑↑↑↑↑↑ 这 4 个被去掉了
```

### 衡量指标

```
ΔSharpe  = Sharpe(NLP-ON) − Sharpe(NLP-OFF)
ΔReturn  = Return(NLP-ON) − Return(NLP-OFF)

ΔSharpe > 0 → NLP 信号有帮助（正面的边际价值）
ΔSharpe ≈ 0 → NLP 信号没什么用
ΔSharpe < 0 → NLP 信号反而有害（引入了噪声）
```

## 8.3 多种子实验

**大白话**：AI 训练有随机性——同一个设置跑两次，结果可能不一样（就像考试有发挥好和发挥差的时候）。多种子实验就是**同一个实验跑多次**，看结果的稳定性。

**本项目的设置**：
```python
DQN_SEEDS = [42, 123, 456]  # 3 个随机种子
```

**对每只股票、每种配置**：
```
Seed 42:   训练 200 episodes → Sharpe = 0.8
Seed 123:  训练 200 episodes → Sharpe = 0.3
Seed 456:  训练 200 episodes → Sharpe = 0.5

平均 Sharpe = 0.53
标准差 σ = 0.21
```

**为什么要这么做**：
1. 排除"运气"因素（也许某次好结果只是运气好）
2. 计算标准差（σ），衡量结果的**稳定性**
3. 统计检验：如果 ΔSharpe < σ，说明差异可能是噪声，不是真实差异

## 8.4 结果解读

### 核心数据

```
DQN 训练标准差（σ）: ±0.3 ~ 0.5 Sharpe    ← AI 自身的不稳定性
NLP 贡献（ΔSharpe）: ≈ 0.05 ~ 0.15         ← NLP 信号的效果
```

### 如何解读

```
┌──────────────────────────────────────────────────┐
│                                                  │
│   σ = 0.4    ┌────────────────────┐              │
│              │    DQN 训练方差     │              │
│              └────────────────────┘              │
│                                                  │
│   Δ = 0.1         ┌────┐                        │
│                    │NLP │                        │
│                    └────┘                        │
│                                                  │
│   0 ──────────────────────────────────── Sharpe  │
│                                                  │
│   结论：Δ << σ → NLP 效果被 DQN 自身方差淹没      │
│                                                  │
└──────────────────────────────────────────────────┘
```

**大白话**：DQN 每次训练的"发挥波动"是 ±0.3~0.5，而 NLP 信号带来的改善只有 0.05~0.15。就像一个篮球运动员投篮——他的命中率本身就上下波动 30%~50%，你给他换了一双"稍微好一点"的球鞋，提升了 5%~15% 的命中率——这个提升太小了，**无法确认到底是球鞋的作用还是恰好今天手感好**。

### 这不是"失败"

这个结果在学术上非常有价值，因为它**量化了 NLP 信号边际价值的上限**：

1. **明确了边界条件**：在 200 episode 训练规模下，NLP 信号无法产生统计显著的改善
2. **指出了改进方向**：要么增大训练规模（更多 episode），要么改进情感分析方法
3. **避免了过度声称**：很多论文声称"NLP 对交易有帮助"，但没考虑 DQN 自身的方差——我们更诚实

### 模拟实盘 P&L ≈ −$99 的解释

```
总交易次数: 约 19 笔
每笔交易成本: 0.1%
累计交易成本: 约 $1,900 × (平均仓位比例) ≈ $99-100

结论：模拟实盘接近零收益 ≈ 收益刚好覆盖交易成本
这不是"亏损"，而是"盈亏平衡"
```

# 第九章 模拟实盘（Paper Trading）

**负责文件**：`paper_trader.py`

## 9.1 什么是模拟实盘

**大白话**：用训练好的 DQN 模型在**新的、未见过的数据**上做交易，但不用真钱——只是模拟记录"如果真做了这些交易，会赚多少/亏多少"。

**跟训练的区别**：
- **训练阶段**：Agent 在学（允许犯错，通过错误学习）
- **模拟实盘阶段**：Agent 已经学好了，用学到的策略做实际交易（不允许再修改策略）

## 9.2 信号生成过程

**`PaperTrader.generate_signal()` 的工作流程**：

```
步骤 1：获取当前市场的状态向量（跟训练时一样的 11 维特征）
步骤 2：把状态向量输入训练好的 DQN 网络
步骤 3：网络输出 3 个 Q 值（Buy / Sell / Hold 的期望价值）
步骤 4：选择 Q 值最大的动作作为交易信号
步骤 5：返回：动作 + 3 个 Q 值 + 置信度
```

### 什么是置信度

**大白话**：Agent 对自己的决策有多大信心。

```python
# 简化计算：
Q_values = [Buy: 0.15, Sell: -0.03, Hold: 0.12]
max_Q = 0.15 (Buy)
置信度 = softmax(max_Q) 或 max_Q / sum(|Q_values|)

# 置信度高：最优动作明显好于其他选择 → 比较确定
# 置信度低：三个动作的 Q 值差不多 → 不太确定
```

**为什么需要置信度**：
- 高置信度信号 → 可以执行
- 低置信度信号 → 可能应该观望（不操作）

## 9.3 Q-values 和置信度

**完整示例**：

```
AAPL 在 2026-05-15 的信号生成：

状态向量: [1.03, 1.01, 1.08, 0.15, 0.002, 0.6, 0.4, 0.5, 0.3, 0.2, 0.15]
  ↓ DQN 推理
Q-values: [Buy=0.15, Sell=-0.03, Hold=0.12]
  ↓
最优动作: Buy (Q=0.15 最大)
置信度: 高 (Buy 明显优于其他)
  ↓
交易信号: 买入 AAPL，约投入当前现金的 10%
```

```
TSLA 在 2026-05-15 的信号生成：

状态向量: [0.95, 0.97, 0.88, -0.20, -0.008, 0.3, 0.7, -0.4, -0.2, -0.3, 0.4]
  ↓ DQN 推理
Q-values: [Buy=-0.05, Sell=0.02, Hold=0.01]
  ↓
最优动作: Sell (Q=0.02 最大，虽然很小)
置信度: 低 (三个 Q 值都很接近 0)
  ↓
交易信号: 卖出少量 TSLA（或因为置信度太低而跳过）
```
